Add-on Presentation

Present add-ons in a way that describes their functionality in clear terms. This involves choosing a descriptive name for the add-on, including demonstrative screenshots, and making the description understandable. For bonus points, try to make the description conform to Mozilla’s copy tone.

Images

Keep your images up to date. If they show OS elements make sure it is a current version of a well known OS (such as the latest versions of Windows or Mac OS X). This will reassure your potential users that this add-on is up to date.

Installation

On installing an add-on the user will see a panel indicating the add-on has been installed. This panel will be placed underneath the icon of the add-on. If the add-on does not have a visual representation, the panel will be placed underneath the menu button.

Browser Action

(related to chrome.browserAction)

Use browserActions to put a button in the main Mozilla Firefox toolbar. Per default this button will be placed to the right of the address bar. However, the user is free to move this button to multiple locations in the browser. In addition to its button, a browserAction should also have a tooltip, and can have a badge, a panel and can be highlighted.

Button

A browserAction button is represented by an icon. Its states are defined by Firefox and will be adapted to match the operating system. Besides those states a button can be highlighted to indicate important events.

Icon

The icon should be clear and distinguishable in one color. To keep a clean appearance within Firefox and match individual OSs we will color the SVG provided in the same color as other toolbar icons. The icon should be 16×16px outline SVG. For compatibility with Chrome/Opera we will also support full color images but strongly recommend matching your icon’s color with native icons. States

Browser actions as other buttons have different appearances indicating their state.

enabled Basic state of a button if it’s function is available.
disabled Basic state of a button if it’s function is not available.
Should not be needed on a browserAction as they should work all the time. If it does not, consider if it should rather be a pageAction.
hover (if not disabled) Controlled by Firefox. Can not be influenced by an add-on.
active (if not disabled) Controlled by Firefox. Can not be influenced by an add-on.
highlighted To indicate an important active state of a browserAction you can set it to be highlighted. This will color the icon blue until the state is resolved. (e.g. set Bookmark or finished Download)
Can only be used if the icon is provided as an SVG

Locations

A browserAction by default is placed in the toolbar to the right of the address bar. The user can however move it to any other location in the toolbar, the bookmark toolbar, or the menu panel. In the menu panel the content specified to be displayed in the popup will be displayed.

  • in Toolbar (default)
  • in Menu Panel
  • in Bookmark Toolbar

Tooltip

Users should be able to understand what will happen when they click the button. Therefore the tooltip will describe what the button does by clearly stating the action that will be performed on click. (Do not just put the name of the add-on in the tooltip, unless it is very descriptive of the action the button will perform!)

Badge

Use sparingly to inform users about important events.

Can be up to 4 characters and will not extend beyond the borders of the button. We recommend using one of four states provided for different severity of notifications: Info (grey #ebebeb), Casual (blue #0996f8), Warning (red #d92215), and Update (green #57bd35).

For compatibility with Chrome/Opera we will also support free color selection, recommend however usage of Firefox defaults to keep consistency.

Panel

Use a panel if your browserAction provides the user with multiple actions or displays an other form of content on request. To best integrate with Firefox make sure to align the content of the panel with our recommended styles.

Tips

For the best visual impact, follow these guidelines:

  • Use browserActions for features that make sense on most pages.
  • Don't use browserActions for features that make sense for only a few pages.
    Use pageActions instead.
  • Use SVG icons that blend well with the other icons in Firefox.

Page Action

pageActions are like browserActions, but with a 12x12 icon.
