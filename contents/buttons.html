Buttons and Links

Buttons

Use these buttons to indicate actions in the main pages of the application (anywhere that is not part of or attached to the browser chrome). The default button color is blue; use green and red for buttons when . . . When designing for Windows, the native OS will place the primary (or default) action to the left of the secondary action. On Mac OS and in most websites and web applications, the primary (or default) action appears to the right of the secondary action.

<button>Button</button>
<button class="default">Button</button>

OS X Buttons

Buttons on OS X and later platforms will adopt a 4px border radius.

Combo Buttons

Use discrete combo buttons to show separate (and often opposing) but contextually related items (like Donâ€™t Allow and Allow). Use joined combo buttons to indicate actions in a multi-step processes (like Copy + Cut + Paste or Increase + Reset + Decrease Zoom Level).

Default Footer Buttons

Use footer buttons at the bottom of panels or in-content dialogs. When used alone, as in the Show All Downloads panel, use grey. When two or more footer buttons appear together, use blue to indicate the primary action and grey, the secondary action.

Disclosure Button

Use disclosure buttons to allow users to display and hide additional content in the interface.

Drop Down Buttons

Use dropdown buttons to display predefined lists of actions or choices. Use a single dropdown in conjunction with a submit action. Use combo dropdowns when there is a predefined list of primary actions and a predefined list of secondary actions. Place the button for the initial action on the left and a dropdown of secondary actions on the right. Secondary actions chosen from the dropdown submit immediately. (Note: Avoid obscuring options that users may not anticipate or understand. In these cases, it may be better to expose options through radio buttons.)

<select> <option value="value1" selected>Dropdown</option> <option value="value2">List Item</option> <option value="value3">List Item</option> </select>

Directional Button

Use directional buttons to allow users to advance through steps in a process or access previous or subsequent pages in a result or process.

Links

Use text hyperlinks as navigation to guide users to a new page in the application or to anchor users to a section within a page. (Note: In Firefox, buttons and links have clear and distinct purposes. Use buttons to indicate user actions that result in an immediate change to the application upon click.)
